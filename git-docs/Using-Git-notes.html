Lesson 1

git commands discussed

  • git log
  • git log --stat
  • git diff id1 id2
  • git clone url
  • git config key value
  • git config --global key value
  • git checkout id

Notes

  • Use git log to view commit history for a repository. Note the ID (the long hexadecimal value), which uniquely identifies each commit.

  • Use git diff id1 id2 to see the difference between two commits with the given id values.

  • Use one commit per logical change

  • A commit is a snapshot of every file in the repository at the time of the commit.

  • git log --stat shows a log along with some information about what files have changed.

  • git clone url creates a clone of a given repository (identified by the url) to your local machine.

  • git config key value can be used to set git configuration values for a particular repository.

  • git config --global key value can be used to set git configuration values system-wide.

  • git checkout id switches the files in your working directory to the version(s) associated with the given id.

  • git-completion.bash, git-prompt.sh, and .bash_profile customize the bash environment.
