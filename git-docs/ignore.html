Telling git to ignore certain file types

You can create a file of globs (command line file patterns), and git will ignore any files with names matching any of those globs. There are a number of places this file can be placed:

  • .../.git/info/exclude
  • .gitignore

Place the glob expressions one per line in the file. Lines beginning with a hash (#) will be ignored as comments. The following would cause git to ignore files ending in .o, .a, and ~:

      *.[oa]
      *~
